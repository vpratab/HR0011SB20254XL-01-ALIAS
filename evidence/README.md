# Evidence Guide

The `evidence/` directory contains deterministic outputs generated by `./scripts/run_scenarios.sh`.

## Scenario Inventory

| Scenario | Directory | Purpose |
| --- | --- | --- |
| [Nominal Fireline Recon Leg](./scenario_01_nominal_response_leg/trust_scorecard.json) | `scenario_01_nominal_response_leg` | Emergency autonomy executes a stable wildfire reconnaissance leg with healthy standoff, clearance margin, and recovery readiness. |
| [Smoke Corridor Compression](./scenario_02_hazard_compression/trust_scorecard.json) | `scenario_02_hazard_compression` | Fireline hazard margins and air-ground coordination begin to degrade but remain within recoverable limits. |
| [Unrecoverable Fireline Divert](./scenario_03_unrecoverable_emergency_path/trust_scorecard.json) | `scenario_03_unrecoverable_emergency_path` | Divert reachability, hazard margin, and mission-app recovery readiness collapse simultaneously, forcing a reject-grade mission assurance event. |

## Files Per Scenario

- `input_log.jsonl`: scenario input trace used by the runtime monitor
- `timeline.json`: frame-by-frame trust and property outcomes
- `trust_scorecard.json`: summary counts and final trust score
- `proof_log.txt`: concise textual ledger of non-accept events
- `trace.svg`: lightweight trust-over-time visualization
- `summary.json`: compact machine-readable scenario summary

## Recommended Review Path

0. read `scorecard_summary.md` for the cross-scenario result table
1. open `trust_scorecard.json`
2. inspect `timeline.json` for the first non-accept frame
3. inspect `proof_log.txt` for the short event ledger
4. open `trace.svg` for a quick visual read
- optional browser inspection: `../tooling/viewer/index.html`
